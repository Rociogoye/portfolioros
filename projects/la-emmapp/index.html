La EmmApp

Jul 14, 2026 min read

The project

La EmmApp is an active web application created to bring together the most important information for one or more pets in one place. It lets users check basic details, vaccines, medication, and upcoming appointments through a cleaner, more current interface.

The project started as part of my web training and I approached it like a product: with a clear problem, a defined user, and a concrete need to simplify daily care management.

Open La EmmApp · View the code on GitHub

Context and problem

  • User: people who need to keep a pet’s health and care under control.
  • Need: bring data, reminders, and appointments into one place.
  • Problem: the information is usually spread across records, notes, and memory.

My role

  • Designed and developed the full application.
  • Made decisions about data structure, flows, and information presentation.
  • Defined the interface so the main view solved the most frequent task.
  • I keep iterating on the experience, code, and visual clarity.

Working process

  1. Identified the problem and the main use case.
  2. Designed the information structure and core flows.
  3. Built the interface and main screens.
  4. Connected registration, editing, and review features.
  5. Reviewed responsive behavior, visual state, and publishing.

Strategic decisions and UX

  • Prioritized the pet overview instead of secondary screens.
  • Organized health information into understandable blocks.
  • Reduced visual noise so the app is quick to understand.
  • Kept the current interface cleaner than the earlier screenshot in the portfolio.

Features

  • Sign up and login with email or Google.
  • Create, view, edit, and delete pets.
  • Image upload.
  • Vaccine and medication tracking.
  • Vet and grooming appointment management.
  • Basic alerts and health states.
  • Pipi-can map on the homepage.

Problems and solutions

  • Problem: health information was scattered.
  • Solution: centralize it in clear pet cards and views.
  • Problem: too many tasks in one app.
  • Solution: prioritize the most frequent navigation and simplify the main screen.
  • Problem: needed a living base.
  • Solution: keep a published version and continue iterating on UI and experience.

Tools

The project uses HTML, CSS, JavaScript, PHP, MySQL, PDO, and Composer. It was developed locally with MAMP, uses GitHub for version control, and has a live public version.

Result and impact

  • App published and accessible online.
  • Interface designed for real use, not just as an academic demo.
  • Clear proposition to keep vaccines, medication, and appointments up to date.
  • Useful example of product thinking, iteration, and current technical judgment.

Visual evidence

The main image shows the real published hero of the application, with the current hierarchy and the three main functional blocks visible in the first screen.

Evolution

  • Continue simplifying forms.
  • Improve accessibility and state messages.
  • Review the mobile experience.
  • Keep expanding the available reminders and resources.
