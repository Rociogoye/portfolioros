La EmmApp

14 jul 2026 min de lectura

El proyecto

La EmmApp es una aplicación web activa creada para reunir en un solo lugar la información importante de una o varias mascotas. Permite consultar datos básicos, vacunas, medicaciones y próximas citas mediante una interfaz visual más limpia y actualizada.

El proyecto nació como parte de mi formación web y lo fui tratando como un producto: con un problema claro, un usuario definido y una necesidad concreta de simplificar la gestión diaria.

Abrir La EmmApp · Ver el código en GitHub

Contexto y problema

  • Usuario: personas que necesitan tener bajo control la salud y los cuidados de una mascota.
  • Necesidad: concentrar datos, recordatorios y citas en un único lugar.
  • Problema: la información suele quedar repartida entre cartillas, notas sueltas y memoria.

Mi rol

  • Diseñé y desarrollé la aplicación completa.
  • Tomé decisiones sobre estructura de datos, flujos y presentación de la información.
  • Definí la interfaz para que la vista principal resolviera la tarea más frecuente.
  • Sigo iterando sobre la experiencia, el código y la claridad visual.

Proceso de trabajo

  1. Identificación del problema y del uso principal.
  2. Diseño de la estructura de información y de los flujos base.
  3. Desarrollo de la interfaz y de las pantallas principales.
  4. Conexión de funcionalidades de registro, edición y consulta.
  5. Revisión de responsive, estado visual y publicación.

Decisiones estratégicas y UX

  • Dar prioridad a la vista de mascotas y no a pantallas secundarias.
  • Ordenar la información sanitaria por bloques comprensibles.
  • Reducir ruido visual para que la aplicación se entienda rápido.
  • Mantener una interfaz más limpia en la versión actual frente a la antigua captura del portfolio.

Funcionalidades

  • Registro e inicio de sesión con correo electrónico o Google.
  • Creación, consulta, edición y eliminación de mascotas.
  • Subida de imágenes.
  • Registro de vacunas y medicaciones.
  • Gestión de citas veterinarias y de peluquería.
  • Alertas y estados de salud básicos.
  • Mapa de pipicans en la página principal.

Problemas y soluciones

  • Problema: la información sanitaria estaba dispersa.
  • Solución: centralizarla en fichas y vistas claras por mascota.
  • Problema: demasiadas tareas en una misma app.
  • Solución: priorizar la navegación más frecuente y simplificar la vista principal.
  • Problema: necesidad de mantener una base viva.
  • Solución: dejar una versión publicada y seguir iterando sobre interfaz y experiencia.

Tecnologías

El proyecto utiliza HTML, CSS, JavaScript, PHP, MySQL, PDO y Composer. Fue desarrollado en un entorno local con MAMP, utiliza GitHub para el control de versiones y cuenta con una versión pública activa.

Resultado e impacto

  • Aplicación publicada y accesible en línea.
  • Interfaz pensada para uso real, no solo como demo académica.
  • Propuesta clara para mantener al día vacunas, medicación y citas.
  • Caso útil para demostrar product thinking, iteración y criterio técnico actual.

Evidencia visual

La imagen principal muestra el hero real de la aplicación publicada, con la jerarquía actual y los tres bloques funcionales principales visibles desde el primer pantallazo.

Evolución del proyecto

  • Continuar simplificando formularios.
  • Mejorar accesibilidad y mensajes de estado.
  • Revisar la experiencia en dispositivos móviles.
  • Seguir ampliando recursos y recordatorios disponibles.
